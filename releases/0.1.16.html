Volt 0.1.16

The scopes-only fix wasn't enough: any normalization gap between the stored
credential and the editable draft (a session key saved with a trailing newline,
nil vs empty OAuth scopes/strings) made a pristine load look like an edit.

Route both the draft and the saved baseline through one canonical form
(ClaudeCredentials.canonical / OpenAICredentials.canonical) that trims
whitespace and treats empty strings/arrays as absent. Because both sides
normalize identically, dirty state is true only when the user actually changes a
value. Add CredentialDirtyStateTests to lock the load round-trip invariant,
including the browser-session and trailing-whitespace cases.
